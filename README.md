# AIRegistry

Lightweight registry service for AI tools and MCP services.

It exposes CRUD APIs and service discovery by both UUID and service name.

## Features

- Register MCP services and tool metadata
- Fetch by UUID: `GET /registries/{id}`
- Fetch by name: `GET /registries/by-name?name=...`
- List all registries: `GET /registries`
- Update and delete entries

## Tech Stack

- Go 1.26
- Chi router
- PostgreSQL
- pgx/v5 connection pool

## Configuration

The service reads environment variables from `.env`.

Required variables:

```env
DBHOST=127.0.0.1
DBPORT=5432
DBUSER=cfgmgr
DBPASSWORD=cfgmgrpass
DBNAME=cfgmgr
SSLMode=disable
REGISTRYMGR_LISTENING_PORT=9098
```

## Database Setup

Create the `registries` table:

```sql
CREATE TABLE registries (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name          TEXT NOT NULL,
    type          TEXT NOT NULL,
    endpoint      TEXT NOT NULL,
    protocol      TEXT NOT NULL DEFAULT 'http',
    auth_required BOOLEAN NOT NULL DEFAULT false,
    policy_ids    TEXT[] NOT NULL DEFAULT '{}',
    metadata      JSONB NOT NULL DEFAULT '{}',
    status        TEXT NOT NULL DEFAULT 'active',
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_registries_name ON registries (name);
CREATE INDEX idx_registries_type ON registries (type);
CREATE INDEX idx_registries_status ON registries (status);
```

If `gen_random_uuid()` is unavailable in your PostgreSQL version, use:

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

and replace `gen_random_uuid()` with `uuid_generate_v4()`.

## Run Locally

```bash
go mod tidy
go run .
```

Service starts on `http://localhost:9098` when `REGISTRYMGR_LISTENING_PORT=9098`.

## Run with Docker Compose

```bash
docker compose up -d --no-deps --force-recreate airegistry
```

Note: current compose file sets `DBHOST=db`, so ensure a PostgreSQL container/service named `db` is reachable on the same network.

## API

### 1) Create Registry

`POST /registries`

```json
{
  "name": "neo4j-mcp",
  "type": "mcp_service",
  "endpoint": "http://localhost:9094/mcp",
  "protocol": "http",
  "auth_required": true,
  "policy_ids": [],
  "metadata": {
    "env": "local",
    "owner": "data-team",
    "auth": {
      "type": "basic",
      "username": "neo4jUser",
      "password": "neo4jPass"
    }
  },
  "status": "active"
}
```

### 2) Get by Name

`GET /registries/by-name?name=neo4j-mcp`

### 3) Get by ID

`GET /registries/{id}`

### 4) List All

`GET /registries`

### 5) Update

`PUT /registries/{id}` with the same JSON shape as create.

### 6) Delete

`DELETE /registries/{id}`

## Notes

- `id`, `created_at`, and `updated_at` are generated by DB/service during creation.
- Keep credentials in metadata secure; avoid storing production secrets in plain text.
